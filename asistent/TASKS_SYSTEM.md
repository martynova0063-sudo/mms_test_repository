# Система отслеживания активных задач

## Обзор

Добавлена система отслеживания задач регистрации в реальном времени. Теперь вы видите не только исторические статусы (которые были несколько часов назад), но и текущие задачи, которые выполняются прямо сейчас.

## Что изменилось

### 1. Новые файлы

- **`task-manager.js`** - Модуль управления задачами в реальном времени
  - Хранение активных задач в памяти
  - SSE (Server-Sent Events) для push-уведомлений
  - API для создания, обновления и завершения задач

### 2. Обновлённые файлы

- **`app.js`**
  - Интеграция с `task-manager.js`
  - Новый endpoint `GET /api/tasks` - получение всех задач (активные + исторические)
  - Новый endpoint `GET /api/tasks-stream` - SSE стрим для задач в реальном времени
  - Обновлён `/start-registration` для создания задач

- **`registration-agent.js`**
  - Добавлены параметры `taskIds` и `taskCallbacks` в `runRegistration()`
  - Обновление статуса задачи во время выполнения регистрации
  - Завершение задачи с финальным статусом (success/error)

- **`public/index.html`**
  - Автоматическое подключение к SSE при открытии панели задач
  - Отображение прогресс-бара для активных задач
  - Отображение текущего шага выполнения
  - Авто-обновление при получении событий от сервера

## Как это работает

### Поток выполнения

1. **Запуск регистрации**
   ```
   Пользователь → POST /start-registration → Создание задач → taskManager.createTask()
   ```

2. **Выполнение регистрации**
   ```
   RegistrationAgent → taskCallbacks.updateStatus() → Обновление статуса → SSE broadcast
   ```

3. **Завершение задачи**
   ```
   RegistrationAgent → taskCallbacks.completeTask() → Завершение → SSE broadcast → Удаление через 30 сек
   ```

### Статусы задач

| Статус | Описание |
|--------|----------|
| `queued` | Задача в очереди на выполнение |
| `active` | Задача выполняется (с прогрессом и текущим шагом) |
| `captcha_pending` | Ожидает решения капчи |
| `success` | Регистрация успешна |
| `error` | Произошла ошибка |

### SSE события

Сервер отправляет события в формате:

```json
{
  "type": "init",
  "tasks": [...]
}
```

или

```json
{
  "type": "update",
  "task": {
    "tempId": "task_1234567890_0",
    "website": "https://example.com",
    "company": "Компания",
    "catalog": "https://catalog.ru",
    "status": "active",
    "progress": 45,
    "currentStep": "Заполнение формы",
    "created_at": "2024-01-01T12:00:00.000Z"
  }
}
```

## API Endpoints

### GET /api/tasks
Получить все задачи (активные + исторические из БД)

**Ответ:**
```json
{
  "tasks": [...],
  "count": 10
}
```

### GET /api/tasks-stream
SSE стрим для получения обновлений задач в реальном времени

**Использование:**
```javascript
const eventSource = new EventSource('/api/tasks-stream');
eventSource.onmessage = (event) => {
  const data = JSON.parse(event.data);
  if (data.type === 'init') {
    // Инициализация с текущими задачами
  } else if (data.type === 'update') {
    // Обновление конкретной задачи
  }
};
```

## Использование в интерфейсе

1. Откройте панель **"Активные задачи"** (кнопка с иконкой списка)
2. Нажмите **"🔄 Обновить"** для загрузки текущего состояния
3. Задачи будут автоматически обновляться в реальном времени
4. Фильтруйте задачи по статусу через выпадающий список

## Пример отображения

Для активных задач показывается:
- 🔵 Индикатор статуса (пульсирует для активных)
- 📍 Текущий шаг выполнения
- 📊 Прогресс-бар (0-100%)
- ⏱️ Время с момента создания
- ⚠️ Ошибка (если есть)

## Технические детали

- Активные задачи хранятся в памяти сервера
- Задачи автоматически удаляются через 30 секунд после завершения
- SSE автоматически переподключается при разрыве соединения
- Исторические задачи загружаются из БД (без полей progress/currentStep)
